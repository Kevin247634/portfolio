ASU Class Search Website UX Case Study

Problem

ASU class search website is a website for students at ASU to look up for classes and add those classes to their schedule. It is a website full of powerful functionalities that aim to make the process of class searching easier for students. However, there are some problems in the website that decrease the usability, resulting in students might face difficulties and confusion when using the website. Take the consistency on the site as an example, when you click on the view my schedule button, instead of showing your current schedule on the website, it takes you to another website, which is My ASU page. This causes confusion to the users since the whole webpage’s layout is different, and the users will have to familiarize themselves with the website again.

Goals

The goal of this UX case study is to find out problems of the website and identify some potential factors that lower the usability of the website. After identifying those problems and factors, suggestions will be made based on the data I collected through user research.

Method – Survey (User Research)

Overview

In order to find out what else bothers the users of ASU Class Search, a designed survey is used in this research in order to collect more detailed information of how users interact with the ASU class search website. The survey is divided into four sections, demographic question section, experience of using the site section, external link experience and an overview rating section. Each section contains questions regarding different topics, and contains both qualitative ones and quantitative ones.

Solution: User Stories

Goal 1:

To search for classes that interest him.

User Stories 1

To search for classes that interest him. As a student who is interested in engineering , I want to search for classes on the website so that I can find classes that interest me.

Acceptance Criteria 1:

  • The search bar should include a broader list of keywords that one may use to search for interested classes.
  • “Class Quiz” may change to another name so that people can easily understand what this function is for.
  • There should be a list of Subject codes with explanations of what department/ program each represents.

Goal 2:

Find out what his required and elective courses are.

User Stories 2

As a student, I want to have a list of my required and elective courses on the website so that I know what to choose while searching for classes.

Acceptance Criteria 2:

  • Have a section or such for users to look for their department’s requirements.
  • Each class search results with the description of whether it is a required or allowed elective for the user who logged in the site.
  • When users add the class to cart, the system should automatically warn the users if the class does not belong to the department's required/ elective class.
  • A required/ elective courses filter which users can apply when they search.
  • Recommended classes appear when users login to the website.

Goal 3:

Find classes that still have seats for enrollment.

User Stories 3

As a sophomore student, I want to see a list of current open classes so that I can find classes that still have seats for enrollment.

Acceptance Criteria 3:

  • A list of current classes should have a clear notice of whether it is still available for enrollment.
  • The list should be organized with classes that still have seats for enrollment and those that don't have anymore seats.
  • When typing in keywords, classes with seats should pop out first in the search result.

Goal 4:

Acquire detailed information regarding the class one is interested in.

User Stories 4

As a sophomore student, I want to see a syllabus attached to the class so that I can acquire detailed information about the particular class.

Acceptance Criteria 4:

  • A link next to each class that takes you to the syllabus.
  • A detailed description written on the class description so that it’s helpful for me to decide whether I am interested in the class or not.
  • Add a section with past reviews of the class from students.

Goal 5:

Find out what course he is allowed to take.

User Stories 5

As a student, I want to have a course map so that I can find out what course I am allowed to take.

Acceptance Criteria 5:

  • Add a course map list on the page so that users can use it as a reference while searching for classes.
  • Descriptions on each class should include this information.
  • System automatically blocks me from choosing classes that I am not allowed to take.

Goal 6:

Plan courses ahead.

User Stories 6

As a student, I want to have a list that shows the upcoming classes of next semester so that I can plan my schedule ahead.

Acceptance Criteria 6:

  • A list of upcoming classes for next semester is available on the website.

Lesson Learned

User research

One thing I did not do so well in the user research phase was that the survey I created was not inclusive enough, making it harder for me to gather enough data to create personas and user stories. Also, some of my questionnaires in the survey were not clear enough or sort of leading participants toward a specific answer. This is something I definitely have to work on in the future.

Personas

One of the main lessons learned from this part of the project is that the persona’s layout should be the same so that each of the personas contains the same information as another. This is helpful for research since that same layout makes it easier to do comparison. Another thing I noticed is that the needs and goals should be carefully addressed. Since that these two criteria are somewhat similar to each other, making sure that the needs are something that help users to achieve their goals is crucial.

Next Step

For the prototype, I am planning on using “Figma” as my tool for it allows designers to build an interactive prototype. After that, I will take my prototype and perform a usability test on it to find out where to improve, whether it fulfills users goal or not.
