ASU Class Search Website UX Case Study

Problem

ASU class search website is a website for students at ASU to look up for classes and add those classes to their schedule. It is a website full of powerful functionalities that aim to make the process of class searching easier for students. However, there are some problems in the website that decrease the usability, resulting in students might face difficulties and confusion when using the website. Take the consistency on the site as an example, when you click on the view my schedule button, instead of showing your current schedule on the website, it takes you to another website, which is My ASU page. This causes confusion to the users since the whole webpage’s layout is different, and the users will have to familiarize themselves with the website again.

Role

Since this is a personal project, my role in this project is both the ux researcher who conducted research to gather more information and the ux designer who provided possible design solutions to solve the problem.

Goals

The goal of this UX case study is to find out problems of the website and identify some potential factors that lower the usability of the website. After identifying those problems and factors, suggestions will be made based on the data I collected through user research.

Process

User Research

In order to find out what else bothers the users of ASU Class Search, a designed survey is used in this research in order to collect more detailed information of how users interact with the ASU class search website. The survey is divided into four sections, demographic question section, experience of using the site section, external link experience and an overview rating section. Each section contains questions regarding different topics, and contains both qualitative ones and quantitative ones.

Personas


Paul represents the users who want to find the required & allowed elective course on the ASU Class Search website. This persona was actually very helpful to me since the goal of this persona was concluded from many of the participants’ opinions.

Nathan represents people who think the current descriptions given on the result page were not enough and want to see more detailed ones so that they can find out additional information they needed in order to plan his academic life ahead.

George represents the users who want to plan his schedule ahead of time. Since the manage schedule function on current ASU class search website is built on a different website, what this type of users need is a consistent website layout so that it is easier for users to navigate and look up for schedule.

User Stories


Usability Testing

Senario 1:

Assuming you are planning your schedule for next semester. Since you have absolutely no ideas of what classes you should take, you are planning to just go to the ASU class search website and find something out.

Tasks:

  • Find out a required course and an elective that you are allowed to take.
  • Search for the class you choose from the required/ electives list. (If possible . Otherwise choose whatever course you want).
  • Go to the website and check for availability and detailed information of that class.

Senario 2:

After finding out the class, you now have decided to add that class to your cart. However, when you try to enroll in the class, the system tells you that you’re unable to enroll in the class. Thus you have to go back and search for another class.

Tasks:

  • Add the class to your cart.
  • Go to the shopping cart and enroll in the class.
  • Go back to the previous class search result page.

Senario 3:

Finally, you have found another class and finished enrolling in it. You now want to go view your class schedule and see if there is any conflict with your own schedule.

Tasks:

  • Find your class schedule with the website.

Usability Test Result


Comments by Participants:

  • "Class Status" should be defined clearly what is "Open class"? Are those classes that is opened but not available to be added supposed to be listed in open class? When I clicked advanced search and was shocked by too much columns, there's no way go back to original search?
  • Adding a system that can automatically block some class you can't take or at least tell you what course you need a permission in order to enroll.
  • Adding a section where I can find my required classes and electives.

Design Solution- Prototyping

Problem 1: Not able to find the required/ elective course

Solution: Adding a button to the nav bar which links to an entire page containing the user’s required/ elective course list.



Problem 2: Cannot navigate back to the previous class search result page

Solution: Adding a button to search result page so that users can navigate back to the previous page.

The one on the left is the current website. As you can see, the page does not have a link or button that allows users to go back to the previous search result page. This causes some usability issues since if users have more than one classes that they are interested in taking, they would have to keep going back to the homepage and type in same keywords in order to find the classes.

My solution to this is to add a button on the cart page to let users navigate back to previous page if needed.



Problem 3: The Cart, Check my Schedule page has a different layout.

Solution: Instead of using several different websites to provide the function, those functions in the prototype are not embedded in the same website so that the layout now is consistent throughout the site.

These are the two differnt layouts that are currently used on the ASU class search website. The one on the left is mainly used for searching, the right one is used when users are trying to add/ enroll in a class. When users find a class and want to add it to the cart, the website will navigate them from the left one to the right one, which as shown in the research, often casues confusion.


These are the iterations I made. To solve the problem mentioned above, I decided to combine two website into one, unifying the layouts so it's consistent and easier for user to navigate without getting confused.

Lesson Learned

Data Collecting

If there is anything I want to improve upon first, it will be collecting more information through user research. Due to time limitations, I was only able to collect data from six ASU students. However, since there are so many different colleges and students in ASU, I wonder if different colleges’ students could have different results. Perhaps the usability issues found through my participants were not considered problems for students from other colleges, making it harder to integrate issues and prioritize what needs to be improved. I would definitely be interested in coming back to this project and doing some more research in the future.

User research

One thing I did not do so well in the user research phase was that the survey I created was not inclusive enough, making it harder for me to gather enough data to create personas and user stories. Also, some of my questionnaires in the survey were not clear enough or sort of leading participants toward a specific answer. This is something I definitely have to work on in the future.

Personas

One of the main lessons learned from this part of the project is that the persona’s layout should be the same so that each of the personas contains the same information as another. This is helpful for research since that same layout makes it easier to do comparison. Another thing I noticed is that the needs and goals should be carefully addressed. Since that these two criteria are somewhat similar to each other, making sure that the needs are something that help users to achieve their goals is crucial.

Takeaways

What I think I benefited from this project the most is sharpening my skills of using Figam to create a prototype. In order to provide solutions to problems existing on current pages, I forced myself to learn and create a prototype that includes as many details as possible so that when taking it to do the usability test, the participants will be more assimilate into the scenario as if they are actually browsing the website to search for classes. Overall I am pretty satisfied with this case study, it was a project that brought me a lot of fun and I enjoyed the whole process of it.
